# Sidekick Examples

Reproducible example projects for [Sidekick for InDesign](https://exchange.adobe.com/apps/cc/8ebe7f95/sidekick-for-indesign) — the plugin that lets Claude (or any other AI assistant) drive Adobe InDesign.

Each folder is a **self-contained recipe**: a starting InDesign file, any input data, and a `README.md` with the exact prompt you'd give Sidekick and what it should produce. Clone the repo, open the starting file in InDesign, and follow along.

Most folders pair with a write-up on the [Sidekick blog](https://sidekick.eastpole.nl/blog) that walks through the result with screenshots.

## Examples

| Example | What it shows | Blog post |
|---------|---------------|-----------|
| [`pricing-table`](./pricing-table) | Update a proposal's priced table from an Excel file — changed prices, added rows, recomputed total — while keeping the brand font and styling | _(coming soon)_ |
| [`booklet-imposition`](./booklet-imposition) | Reorder a multi-page document into saddle-stitch printer spreads and export a print-ready PDF — the imposition InDesign won't do natively | _(coming soon)_ |

## How to use an example

1. Clone this repo (or use **Code → Download ZIP**).
2. Open the example's starting `.idml` in InDesign.
3. Make sure Sidekick is connected (Claude Desktop running, plugin showing "Connected").
4. Give Claude the prompt from that example's `README.md`.

## A note on the sample files

Everything here is public. To keep it that way safely, every asset is **fictional and self-made** — no real client work, no paid stock, no licensed fonts. If you contribute an example, keep to that bar. See [`CONTRIBUTING.md`](./CONTRIBUTING.md).
